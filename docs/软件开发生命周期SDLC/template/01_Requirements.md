# 📋 需求分析文档 (Lite)

## 1. 项目愿景 (One-Liner)
> 用一句话描述：这是一个给 [谁] 用的 [什么系统]，用来解决 [什么问题]。
> *例：这是一个给个人开发者用的任务管理系统，用来解决多项目进度追踪混乱的问题。*

## 2. 用户角色 (Actors)
- **角色 A** (如：普通用户)：核心诉求...
- **角色 B** (如：管理员)：核心诉求...

## 3. 核心业务流程 (关键！)
*(在此处插入流程图链接或截图)*
> **流程描述**：
> 1. 用户登录 -> 2. 创建任务 -> 3. 分配状态 -> 4. 完成任务 -> 5. 统计报表
     > **异常分支**：
> - 如果任务超时未处理？ -> 自动标记为红色并发送邮件。

## 4. 功能优先级 (MoSCoW 法则)
### 🔴 Must (必须有 - MVP 核心)
- [ ] 功能点 1 (如：用户注册/登录)
- [ ] 功能点 2 (如：任务的增删改查)
- [ ] 功能点 3 (如：状态流转)

### 🟡 Should (应该有 - 重要但可暂缓)
- [ ] 功能点 4 (如：搜索过滤)
- [ ] 功能点 5 (如：数据导出)

### 🟢 Could (可以有 - 锦上添花)
- [ ] 功能点 6 (如：暗黑模式)
- [ ] 功能点 7 (如：动画效果)

### ⚪ Won't (本次不做 - 坚决砍掉)
- [ ] 复杂的权限细分
- [ ] 第三方即时通讯集成

## 5. 非功能需求 (架构师视角)
- **性能**：首页加载 < 1s，支持并发用户数 ~10 (个人够用)。
- **安全**：密码加密存储 (BCrypt)，接口防重放。
- **扩展性**：预留字段 xxx，以便未来支持 xxx。
